# 디자인 협업 플레이북

작성일: 2026-06-08
범위: 범용 원칙 + WSB 브랜드 사이트 구체 적용
1순위 기준: 비주얼·브랜드 차별성 (생성형 AI 티 안 나는, 기억에 남는 디자인)
선호 입력: 레퍼런스 제시 + 형용사·키워드 언어, Claude가 해석
협업 모델: A(레퍼런스 기반 루프) 중심 + 핵심 섹션에 C(병렬 시안) + 토대는 B(디자인 토큰)

## 1. 요청 템플릿

사용자가 디자인을 요청할 때 쓰는 6줄 양식. 다 채울 필요 없고 있는 것만 줘도 된다.

```
[범위]   어느 페이지/섹션 (예: 홈 히어로)
[목표]   이 섹션이 할 일 (첫인상 / 신뢰 / 전환 / 정보전달)
[레퍼런스] URL이나 이미지 2~3개 + "이 부분이 좋다"
[키워드]  형용사 3~5개 (절제된, 산업적, 정밀한, 데이터적)
[제약]   꼭 지킬 것 (브랜드 색·폰트, 데이터 정확성, 카피 톤)
[자유도]  "차용 OK" vs "참고만, 우리 식으로 재해석"
```

`[자유도]` 한 줄이 차별성을 좌우한다. "참고만"이면 패턴만 빌리고 우리 시그니처로 변형한다.

## 2. 협업 루프 (5단계)

1. 번역: 레퍼런스+키워드를 구체적 디자인 언어(색·타이포·간격·레이아웃·모션)로 요약해 한 번 합의. 헛작업을 여기서 거른다.
2. 구현: `frontend-design`으로 빌드.
3. 검증: `node scripts/screenshot.mjs`로 desktop/mobile 스크린샷을 함께 본다.
4. 조정: 스킬 맵으로 핀포인트 수정.
5. 마감: `polish` + `audit`.

핵심 섹션(히어로 등) 1~2개는 루프 대신 C(병렬 시안 3개 → 비교 → 선택/합치기)로 발산한다.

## 3. 스킬 맵

사용자는 자연어로 말하면 되고, 아래는 Claude가 뒤에서 고르는 도구다. 알아두면 콕 집어 요청할 수 있다.

| 상황 | 한마디 | 스킬 |
|---|---|---|
| 새로/크게 디자인 | "이 섹션 새로 디자인" | `frontend-design` |
| 너무 밋밋·약함 | "더 과감하게" | `bolder` |
| 너무 과함·시끄러움 | "톤 좀 낮춰" | `quieter` |
| 색이 부족 | "색 좀 입혀" | `colorize` |
| 레이아웃·리듬 | "배치·간격 정리" | `arrange` |
| 타이포 | "글자 손봐" | `typeset` |
| 마감 디테일 | "디테일 다듬어" | `polish` |
| 단순화 | "군더더기 빼" | `distill` |
| 평가받기 | "이거 어때? 비평해줘" | `critique` |
| 종합 점검 | "전체 점검" | `audit` / `web-design-guidelines` |
| 반응형 | "모바일 손봐" | `adapt` |
| 견고성(오버플로·i18n·엣지) | "깨지는 데 없게" | `harden` |
| 모션·인터랙션 | "움직임 넣어" | `animate` / `overdrive` |
| 시각 비교 필요 | "여러 안 보여줘" | 브라우저 목업 / 스크린샷 |

## 4. 차별성 가드레일

- 레퍼런스는 패턴만 차용, 시그니처는 우리 것. WSB 시그니처: Data Overlay, FDA 매칭표, 정밀 관측 사진톤, 6~8배 임팩트.
- 색·타이포·간격은 토큰(`src/app/globals.css`)으로 잠가 일관성 유지.
- 핵심 섹션 1~2개만 병렬 시안으로 발산, 나머지는 루프로.
- 브랜드 가이드 v1.0 + em-dash 금지 등 글쓰기 스타일 준수.

## 5. 적용 방법

평소에는 따로 의식할 필요 없다. 디자인 작업을 요청하면 Claude가 이 루프를 기본으로 돈다.

명시적으로 강하게 적용하고 싶을 때:
- 새 섹션을 제대로 만들 때: 요청 템플릿으로 보낸다. 레퍼런스나 키워드가 없으면 Claude가 먼저 물어본다.
- 핀포인트 수정: 스킬 맵의 "한마디"를 그대로 쓰거나 `/bolder` `/polish` 같은 슬래시로 직접 호출한다.
- 핵심 섹션 발산: "히어로 시안 3개 보여줘"처럼 C를 명시한다.

워크플로 예시 (자산 도착 후 히어로):
```
[범위] 홈 히어로
[목표] 압도적 첫인상 + 사업/R&D 무게감
[레퍼런스] (시설 실사) + 좋아하는 산업 사이트 2개
[키워드] 절제된, 산업적, 정밀한, 데이터적
[제약] 브랜드 색·Pretendard, 슬로건 카피 고정, 다크 오버레이 위 텍스트
[자유도] 참고만, 우리 식으로 (Data Overlay 시그니처 유지)
```
→ Claude: 디자인 언어 요약 합의 → 시안 3개(C) → 스크린샷 비교 → 선택안 polish → audit.

이 플레이북 자체는 코드가 아니라 합의된 작업 방식이므로 구현 계획(writing-plans)은 만들지 않는다. 다음 디자인 요청부터 바로 적용된다.
